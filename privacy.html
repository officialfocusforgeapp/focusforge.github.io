FocusForge Privacy Policy

Effective date: March 3, 2026

Summary

  • Accessibility is used only to detect foreground app package names for blocking.
  • No typed text, messages, media content, or screen contents are read.
  • Usage Access is optional and used only for insights/projection features.
  • You can disable features or uninstall at any time in Android settings.

1. Information We Collect

1.1 Account and Profile Data

  • Username, internal user ID, account role (Self/Parent/Child).
  • Avatar, profile line, flair, and in-app preferences.

1.2 Blocking and Configuration Data

  • App blocking rules, schedules, severity, selected app groups.
  • Blocked app attempts, focus sessions, achievements.

1.3 AccessibilityService Data

  • Foreground app package name for rule matching.
  • No keystrokes, no messages, no screen scraping.

1.4 Optional Usage Access

  • Per-app foreground duration statistics via Android Usage APIs.
  • Used for top-app insights and usage projections.

1.5 Optional Social / Family Data

  • Friends list, accountability partners, parent/child links.
  • Event metadata for accountability alerts.

1.6 Ads (When Enabled)

  • Ad providers may process identifiers under their policies.
  • FocusForge only requests ad events required for ad-gate flow.

2. How We Use Information

  • Provide blocking, focus sessions, progression, accountability features.
  • Generate insights when Usage Access is enabled.
  • Protect service security and enforce policies.
  • Improve diagnostics and beta testing.

3. Where Data Is Stored

Primary app data is stored on-device unless backend features are enabled.

If backend mode is enabled, relevant data may be stored in Firebase services configured for FocusForge.

4. Sharing of Information

We do not sell personal data. Data is shared only to operate functionality:

  • Google Firebase (authentication, cloud database).
  • Google/AdMob (if rewarded ads enabled).
  • Legal disclosure if required by law.

5. Data Retention

  • On-device data remains until deletion or uninstall.
  • Cloud data retained while account features remain active.

6. Your Choices and Controls

  • Disable Accessibility or Usage Access in Android settings.
  • Modify/remove configuration data in-app.
  • Request deletion via support email.

7. Children and Family Controls

Parents/guardians are responsible for lawful use of family features and required consent for child accounts.

8. Security

We use reasonable administrative and technical safeguards. No method is 100% secure.

9. International Transfers

If cloud services are used, data may be processed in provider infrastructure regions.

10. Changes to This Policy

Updates will revise the effective date on this page.
